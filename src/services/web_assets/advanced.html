💡 How to read these limits:

  • Internal SRAM (Used): The system's fast on-chip data memory. Watch this percentage closely; if it approaches >90%, dynamic allocations (`Box`, `Vec`, string formatting) risk throwing uncatchable out-of-memory heap panics.
  • Main Task Stack Margin: The deep stack safety headroom left for your execution thread loop. If this dropped footprint drops below 2.0 KB, your application task is dangerously close to a hard fault stack overflow crash.
